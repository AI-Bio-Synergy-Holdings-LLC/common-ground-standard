# Contributing

Thank you for reviewing the Common Ground Standard drafts.

This repository is currently in public review mode. The most useful contributions are specific critiques, proposed edits, missing safeguards, evidence requirements, and anti-capture risks.

## What To Contribute

Helpful contributions include:

- identifying vague language;
- proposing stronger no-go conditions;
- adding missing student, worker, community, data, or ecological safeguards;
- identifying public-benefit claims that need evidence;
- proposing clearer evidence requirements;
- identifying places where a powerful actor could capture or dilute the standard;
- suggesting practical first-pilot requirements.

## What Not To Contribute

Do not submit:

- confidential information;
- personal data;
- student records;
- worker or beneficiary records;
- tribal, Indigenous, community, or ecological data not approved for public release;
- private partner or funder information;
- legal advice you are not authorized to share;
- trade secrets;
- materials you do not have permission to contribute.

## Review Paths

The primary review path is the public website form, which does not require a GitHub account:

- [Review Pathway Assistant](https://common-ground-standard.org/review-assistant.html)
- [Website Review Intake](https://common-ground-standard.org/review-feedback.html)

Use GitHub when you want a critique or proposed source change to be public. A GitHub account is
required, and the issue will display your GitHub identity. Choose one of three forms:

- [Review a draft section](https://github.com/AI-Bio-Synergy-Holdings-LLC/common-ground-standard/issues/new?template=draft-section-review.yml)
- [Flag a risk or missing safeguard](https://github.com/AI-Bio-Synergy-Holdings-LLC/common-ground-standard/issues/new?template=risk-and-safeguard.yml)
- [Improve wording or accessibility](https://github.com/AI-Bio-Synergy-Holdings-LLC/common-ground-standard/issues/new?template=wording-and-accessibility.yml)

Pull requests are welcome for precise wording changes. Website submissions, issues, and pull requests
are triaged into the same review program before v0.2.

## Review Etiquette

Please be direct, specific, and evidence-minded.

Strong review sounds like:

> This clause is too weak because it lets an infrastructure actor claim community benefit without a binding governance process. Minimum fix: require affected-party approval, public benefit-routing receipts, and a no-go trigger if retail-bill impacts exceed the agreed threshold.
